# AI Hedge Fund - 使用指南

## 📋 项目概述

这是一个AI驱动的对冲基金概念验证项目，使用多个AI代理协同工作来做出交易决策。

### 🤖 系统包含的代理

系统包含19个专业AI代理，每个都有特定的投资策略：

1. **价值投资专家** - Aswath Damodaran (估值专家)
2. **价值投资之父** - Ben Graham (安全边际)
3. **激进投资者** - Bill Ackman (大胆持仓)
4. **成长投资女王** - Cathie Wood (创新颠覆)
5. **巴菲特的搭档** - Charlie Munger (优质企业)
6. **大空头** - Michael Burry (深度价值)
7. **Dhandho投资者** - Mohnish Pabrai (低风险高回报)
8. **黑天鹅分析师** - Nassim Taleb (尾部风险)
9. **实战投资者** - Peter Lynch (十倍股)
10. **成长投资专家** - Phil Fisher (深度研究)
11. **印度大牛** - Rakesh Jhunjhunwala
12. **宏观传奇** - Stanley Druckenmiller (不对称机会)
13. **奥马哈先知** - Warren Buffett (优质企业)
14. **估值代理** - 计算内在价值
15. **情绪代理** - 分析市场情绪
16. **基本面代理** - 分析基本面数据
17. **技术分析代理** - 分析技术指标
18. **风险管理代理** - 计算风险指标
19. **投资组合经理** - 做出最终交易决策

## 🚀 快速开始

### 1. 安装依赖

```bash
cd ~/PycharmProjects/ai-hedge-fund

# 安装 Poetry（如果未安装）
curl -sSL https://install.python-poetry.org | python3 -

# 安装项目依赖
poetry install
```

### 2. 配置API密钥

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，添加你的API密钥
# 至少需要配置一个LLM API密钥（如OPENAI_API_KEY）
```

**必需的API密钥：**
- `OPENAI_API_KEY` - OpenAI API密钥（用于GPT模型）
- `FINANCIAL_DATASETS_API_KEY` - 金融数据API密钥

**可选的API密钥：**
- `ANTHROPIC_API_KEY` - Anthropic API密钥（Claude模型）
- `GROQ_API_KEY` - Groq API密钥
- `DEEPSEEK_API_KEY` - DeepSeek API密钥
- 等其他支持的LLM提供商

## 💻 命令行使用

### 基本用法

```bash
# 运行AI对冲基金
poetry run python src/main.py --ticker AAPL,MSFT,NVDA

# 指定时间范围
poetry run python src/main.py --ticker AAPL,MSFT,NVDA --start-date 2024-01-01 --end-date 2024-03-01

# 使用本地LLM（Ollama）
poetry run python src/main.py --ticker AAPL,MSFT,NVDA --ollama

# 显示推理过程
poetry run python src/main.py --ticker AAPL,MSFT,NVDA --show-reasoning
```

### 回测功能

```bash
# 运行回测
poetry run python src/backtester.py --ticker AAPL,MSFT,NVDA

# 回测指定时间范围
poetry run python src/backtester.py --ticker AAPL,MSFT,NVDA --start-date 2024-01-01 --end-date 2024-03-01
```

### 参数说明

- `--ticker` - 股票代码（逗号分隔，如：AAPL,MSFT,NVDA）
- `--start-date` - 开始日期（YYYY-MM-DD格式）
- `--end-date` - 结束日期（YYYY-MM-DD格式）
- `--ollama` - 使用本地Ollama LLM
- `--show-reasoning` - 显示AI代理的推理过程
- `--model` - 指定使用的模型（默认：gpt-4.1）
- `--provider` - 指定模型提供商（默认：OpenAI）

## 🔧 配置说明

### 环境变量配置

在 `.env` 文件中配置以下变量：

```bash
# 必需的API密钥
OPENAI_API_KEY=your-openai-api-key
FINANCIAL_DATASETS_API_KEY=your-financial-datasets-api-key

# 可选的API密钥（根据需要配置）
ANTHROPIC_API_KEY=your-anthropic-api-key
GROQ_API_KEY=your-groq-api-key
DEEPSEEK_API_KEY=your-deepseek-api-key
GOOGLE_API_KEY=your-google-api-key
OPENROUTER_API_KEY=your-openrouter-api-key
```

### 模型配置

默认使用 `gpt-4.1` 模型，可以通过命令行参数更改：

```bash
# 使用不同的模型
poetry run python src/main.py --ticker AAPL --model gpt-4o

# 使用不同的提供商
poetry run python src/main.py --ticker AAPL --provider Anthropic
```

## 🔍 故障排除

### 常见问题

#### 1. "Command not found: uvicorn" 错误

**解决方案：**
```bash
cd app/backend
poetry env remove --all
poetry install
```

#### 2. Python版本问题

- **使用Python 3.11**：Python 3.13+ 可能有兼容性问题
- **检查Python版本：** `python --version`

#### 3. 环境变量问题

- **确保 .env 文件存在** 在项目根目录
- **从模板复制：** `cp .env.example .env`
- **添加你的API密钥** 到 .env 文件

#### 4. 端口被占用

如果端口8000或5173被占用：
```bash
# 杀掉现有进程
pkill -f "uvicorn\|vite"
```
